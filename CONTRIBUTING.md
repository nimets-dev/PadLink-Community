# Contributing to the PadLink community

PadLink's application source code is **not** hosted in this repository, so there's no code to
pull request here. But your reports and ideas directly shape the app — here's how to help:

## 🐛 Report a bug
Use the **[Bug Report](https://github.com/nimets-dev/PadLink-Community/issues/new?template=bug_report.yml)**
form. Include your PadLink version, OS, the mode you were using, and any logs.

## 💡 Request a feature
Use the **[Feature Request](https://github.com/nimets-dev/PadLink-Community/issues/new?template=feature_request.yml)**
form, or float the idea in [Discussions](https://github.com/nimets-dev/PadLink-Community/discussions) first.

## 🎮 Request support for a new controller
This is the most impactful contribution. Use the
**[New Controller Support](https://github.com/nimets-dev/PadLink-Community/issues/new?template=controller_support.yml)**
form and include your controller's USB VID:PID, product string, and — if you can — samples of
how its raw HID reports change as you press buttons. That's exactly what we need to add a decode
profile for your pad.

## Etiquette
- Search existing issues and discussions before opening a new one.
- One issue per report. Keep it focused.
- Be kind. We're a small team and we read everything.
